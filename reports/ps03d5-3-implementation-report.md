# PS-03D5-3 Implementation Report — Create Account Backend, Runtime Disabled

## Summary

Implemented the Create Account backend contract and HTTP handler while keeping it **runtime disabled**. The public `POST /api/games/{gameId}/accounts` route still returns HTTP 403. The handler is exported, fully mapped to the OpenAPI contract, and covered by integration tests mounted in a dedicated test-only Express app.

## Runtime Changes

- `POST /api/games/{gameId}/accounts` remains disabled at runtime: returns HTTP 403 with `Account operations are not authorized`.
- The `createAccountHandler` is exported but **not** mounted in the production router.
- When the handler is mounted in tests, it:
  - Accepts `CreateAccountRequest` and a path `gameId`.
  - Returns HTTP 201 with `AccountDetailResponse` (safe DTO, no secrets/passwords/backup codes/customer info).
  - Returns HTTP 409 `DUPLICATE_WARNING` for unconfirmed duplicates, with only field names.
  - Maps `GameNotFoundError` → 404, `InactiveGameError` → 409, `IdentifierConflictError` → 409, `EncryptionError` → 500.
- No schema or migration changes; no runtime Account mutation activation.

### API server test highlights
- Public route 403 with no writes.
- 201 creation returns safe `AccountDetailResponse`.
- Capacity template and Backup Codes are created.
- 404 for missing/deleted game.
- 409 for inactive game.
- 400 for invalid UUID, email, birthDate, empty backup codes, whitespace backup codes, unexpected fields, `statusOverride`.
- Path `gameId` wins over body `gameId`.
- Unconfirmed duplicate → 409 `DUPLICATE_WARNING` with `DuplicateFieldName` enum values, no writes.
- Confirmed duplicate → 201.
- Missing encryption key → 500 with no writes.

## Known Risks

- The test loader (`test-loader.mjs`) is a local workaround for Node ESM resolution of extensionless imports in source files. It is only used in the API server test script. If the project moves to explicit `.ts` imports everywhere, the loader can be removed.
- The frontend `playsyncer` runtime currently shows a pre-existing `useGames must be used inside <GamesProvider>` error (from PS-03D5-2). This is unrelated to the Create Account backend changes and is not addressed in this sub-stage.
- Root `pnpm run build` requires `PORT`/`BASE_PATH` for Vite-based artifacts; the per-artifact builds were run with explicit values.
